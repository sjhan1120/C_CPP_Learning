[EOF(End Of File)]
    의미:   주어진 입력 파일만 갖고 입력을 받을 때 더이상 읽을 수 있는 데이터가 없는 경우. 즉, 파일의 끝일 때 이를 EOF라고 함.
    주의:   파일의 끝까지 읽었다고 EOF가 되는 것이 아니라 끝까지 읽고난 뒤 그 다음 파일을 읽으려 할 때 읽을 데이터가 없다면 EOF가 되는 것.
            시점을 정확히 파악해야 함.
    
    에디터에서는 입력 파일을 따로 생성하여 읽지 않는 한 일반적인 키보드에서는 EOF키가 없어 EOF 상태를 전송 불가능.
    그 대신 키맵이 따로 있는데, 콘솔에서 윈도우의 경우 CTRL+Z를 입력해주면 됨.

[백준10951 해결법]
    -해결법 1 : scanf
        scanf는 int반환값이 존재함. 반환값은 읽어들인 데이터의 개수를 반환함
        예를들어 scanf("%d %d %d, &a, &b, &c); 라고 하고 3개의 변수에 알맞게 입력이 들어왔다면 3을 반환
        만약 데이터를 읽는동안 파일의 끝(EOF)에 도달한 뒤 읽고자 하면 -1을 반환.

        해결책: while(scanf("%d %d", &a, &b) != -1) { ... }
    
    -해결법 2 : cin
        eof()함수를 사용할 경우 파일의 끝에 도달한 뒤 읽고자 한다면 true를 반환함.
            while(!(cin >> a >> b).eof()) { ... }
        주의: eof()가 우리가 원하는 올바른 시점에 true가 되려면 "읽기 시도를 한 후"에 eof검사를 해야함.

        cin>>a>>b이 조건문 안에 있는 경우 연산자 오버로딩에 의해 boo값으로 true 혹은 false값을 반환함.(스트림이 정상상태면 true 아니면 false)
        따라서 cin>>a>>b과정에서 읽기를 실패했을 경우 스트림 상태를 변경하고, 연산자 오버로딩에 의해 반환되는 값이 true에서 false로 바뀌면서 while문을 종료함.
